# Support

Thanks for using **Aefos AI**. Here is how to get help, depending on what you need.

## 📖 First: the manual

Most questions are answered in the **[User Manual](README.md#documentation)**
(PT-BR / EN) — installation, first steps, Chat, Terminal, providers, configuration,
licensing, and a troubleshooting/FAQ section.

## 🐛 Found a bug?

[Open a bug report](../../issues/new/choose). Please:

- search [existing issues](../../issues) first;
- read the short [Submission Terms](TERMS-ISSUES.md) — **do not** post secrets,
  personal data, or proprietary code;
- include your Aefos version, OS, RAD Studio version, steps to reproduce, and
  sanitized logs.

## 💡 Have an idea?

[Open a feature request](../../issues/new/choose).

## ❓ Questions & discussion

Use **[Discussions](../../discussions)** for usage questions, ideas, and general help
(enable Discussions in the repository settings).

## 🔒 Security

**Do not** open a public issue for a security vulnerability. Follow the private
process in **[SECURITY.md](SECURITY.md)**.

## 🧾 Licensing, partnerships, legal

Email **tecsisinfo.com.br@gmail.com** · Official channel: <https://www.pubpascal.dev>

## Scope of support

Aefos is a harness for **third-party** AI CLIs. Issues with the CLIs themselves
(Claude Code, Codex, GitHub Copilot CLI, Gemini), with RAD Studio, or with the
WebView2 runtime should go to those vendors. See [SECURITY.md](SECURITY.md#scope).
